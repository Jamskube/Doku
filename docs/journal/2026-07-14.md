## Fixed
- **Flou lecteur PDF** : le canvas était dimensionné avant l'apparition de la scrollbar verticale → `max-width:100%` rééchantillonnait d'un facteur non-entier. Corrigé par `scrollbar-gutter: stable` (+ plafond DPR).

## Discoveries
- Canvas HiDPI net = backing-store `cssWidth × dpr` **ET** réserver la gouttière scrollbar (`scrollbar-gutter: stable`) ; sinon flou léger par rééchantillonnage non-entier.
- Éditeur CM6 unique partagé entre onglets : toute insertion faite **après un `await`** doit re-vérifier `app.activeId === tabId` avant `dispatch`, sinon écriture dans le doc affiché entre-temps.
- Écritures pilotées par event (collage) : sérialiser (chaîne de promesses) + boucle de nom unique, sinon deux events rapprochés voient le même nom libre et `rename` (remplace sur Windows) écrase le premier.
- Presse-papier : extraire `getAsFile()` **synchroniquement** (avant tout `await`) — `clipboardData.items` devient inerte après ; le `File` survit, pas le `DataTransferItem`.

## Decisions
- Association OS `.pdf` = `role: "Viewer"` (distinct de `Editor` md/html) ; mécanisme NSIS identique aux `.md/.html` (8.2) → validée « sur confiance », confirmation au prochain build+install.
- `fs:allow-exists` élargi à `**` (cohérent ADR-0005 : read/write/rename/read-dir déjà `**`) plutôt que contourner via `readDir`.
- Nettoyage du tmp-orphelin (`.doku-tmp`) écarté : exigerait `fs:allow-remove` en `**` (supprimer-n'importe-quoi) pour un orphelin rare et masqué de l'explorateur — dette acceptée.
