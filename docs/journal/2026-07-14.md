### Discoveries (Epic 13)
- **Ollama ARM64** (`ollama-windows-arm64.zip`, ~16 Mo) = `ollama.exe` **+** un dossier `lib/ollama/` (DLLs ggml + `llama-server.exe`) chargé relativement au dossier de l'exe ; `externalBin` ne copie que l'exe → `OLLAMA_LIBRARY_PATH` (dev: `CARGO_MANIFEST_DIR/binaries` ; release: `resource_dir()` + `bundle.resources`).
- **Kill d'arbre** : `ollama.exe` spawne `llama-server.exe` en grand-enfant ; `kill`/`taskkill` du parent le laisse orphelin. Un **Job Object `KILL_ON_JOB_CLOSE`** tue tout l'arbre à la mort du process (même crash), grand-enfant hérité inclus.
- **Egress Ollama** : `OLLAMA_NO_CLOUD=1` coupe le poll `model_recommendations` → `ollama.com` (seule sortie autonome de `serve`) ; pas de télémétrie sinon ; clé SSH jamais transmise par `serve`. Pare-feu = garantie dure.
- **Annulation `generate`** : passer un `AbortSignal` au `fetch` ; l'abort coupe le flux client ET serveur (Ollama stoppe à la déconnexion) < 500 ms, texte partiel rendu.
