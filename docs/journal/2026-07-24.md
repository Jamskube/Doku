## Discoveries
- **granite-embedding:278m bat bge-m3 sur tous les axes** (qualité FR, vitesse ×3, RAM ÷2) — mesuré au spike ; la réputation multilingue ne se traduit pas en avantage. BM25 baseline sérieuse (0,867) mais fusion RRF sans gain sur le modèle retenu.
- **Appariement bin/meta après crash** : deux écritures atomiques ne garantissent pas le couple → checksum du bin dans meta.json (leçon AGENTS.md).
- **Un PDF scanné sans entrée meta est ré-indexé en boucle** (le diff le voit « nouveau » à chaque refresh) → entrée marqueur `chunks:[]`. Trouvé par le critic (BLOCK 18.3).
- **`extractPdfText` LÈVE l'AbortError** (contrairement à `chat()`/`generate()` qui l'avalent) → tout appelant a besoin d'un garde `signal.aborted` explicite dans son catch (sinon Stop = fausse carte d'erreur).
- `pdf.ts` initialise son worker au niveau module → browser-only, non importable sous vitest → logique pure séparée dans un module sans pdfjs.

## Decisions
- Sprint RAG isolé (S13) puis sprint dette PDF (S14) — « réutiliser, pas rouvrir » : les stories PDF se branchent sur map-reduce 14.2, index éphémère 15.3, index dossier 15.2 sans les modifier.
- Base vectorielle tierce rejetée (brute-force 2,25 ms/1000 chunks, YAGNI) ; fusion RRF différée ; repli bge-m3 mesuré et prêt (ADR-0015).
- Signal de changement PDF = `stat` (`size:mtime`), pas hash d'octets : compromis coût/justesse assumé (réédition à size+mtime identiques ratée, documentée).
- OCR hors scope : PDF scanné signalé honnêtement, jamais de faux texte (FR-4).
